# Checklist: Collaboration and Handoffs for Expanded Project Personas

Use this checklist to ensure clear responsibilities and smooth handoffs between core and new project roles.

## General

- [ ] Roles and expectations are documented and accessible
- [ ] Project charter or kickoff includes all relevant personas

## UX Designer

- [ ] UX is involved before key feature design decisions are finalized
- [ ] Usability testing results incorporated into backlog refinement
- [ ] Accessibility reviews completed during development

## Release Manager

- [ ] Release plan is communicated to all stakeholders in advance
- [ ] Release criteria and Go/No-Go checklist agreed with PM, Dev, QA, and Support
- [ ] Incident/rollback plan is documented for each release

## Technical Writer

- [ ] Documentation requirements captured as backlog items or acceptance criteria
- [ ] Release notes drafted before production rollout
- [ ] Docs reviewed with Product, Dev, and QA before signoff

## Coordination

- [ ] Roles collaborate on sprint planning for workload awareness
- [ ] Regular reviews or standups include all function leads
- [ ] Retrospectives address pain points in handoffs between roles

---
This checklist is meant to be adapted to specific project needs or phases.
